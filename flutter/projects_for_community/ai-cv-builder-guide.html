cv-rules.md 
# Professional Resume Writing Rules

## 1. THE 6-SECOND RULE
Recruiters spend 6-9 seconds on initial scan.
- Most important info in top third
- Bullets scannable (first 3-4 words only)
- Keyword/metric FIRST in each bullet
- Max 1 page for interns/fresh graduates
- Generous white space

## 2. SECTION ORDER (Student/Intern)
1. Header (Name + Contact)
2. Professional Summary (2-3 sentences MAX)
3. Education (students put HIGH)
4. Experience (reverse chronological)
5. Projects (only if relevant)
6. Technical Skills (hard skills ONLY - categorized)
7. Additional (awards/competitions - if space)

## 3. X-Y-Z BULLET FORMULA (Google Standard)
"Accomplished [X] as measured by [Y], by doing [Z]."
- X = Accomplishment
- Y = Measurable impact (number, %, time saved)
- Z = Technical method
Target: 1 line per bullet, max 2 lines. Under 20 words ideal.

## 4. WORDS TO NEVER USE
Empty claims: Hard worker, Dedicated, Passionate, Detail-oriented, Self-starter, Fast learner, Results-driven, Team player
Weak verbs: Responsible for → Designed, Assisted → Collaborated, Handled → Managed, Worked on → Developed
Filler: "Proven track record", "Strong communication", "References available", "Successfully", "Various", "Etc."

## 5. STRONG ACTION VERBS
Technical: Architected, Engineered, Designed, Developed, Built, Implemented, Automated, Deployed, Optimized
Problem-solving: Resolved, Debugged, Streamlined, Eliminated
Leadership: Led, Spearheaded, Directed, Mentored, Pioneered
Analysis: Analyzed, Validated, Evaluated
Rule: Never repeat same verb twice.

## 6. TAILORING RULES
- Extract exact phrases from JD (ATS is literal)
- Use same phrasing as JD
- Place keywords in: Summary, Skills, first 3 words of bullets
- Never emphasize skill you can't defend in 30-min interview

## 7. SOFT SKILLS RULE
NEVER list soft skills standalone. PROVE through bullets.
Example: "Led 3-person team to deliver app in 2-hour sprint, winning 3rd place"

## 8. FORMATTING
- Single-column layout (ATS compatible)
- Standard headings: "Experience", "Education", "Skills"
- Font: Arial, Calibri, Roboto - 10-12pt body, 14-16pt headers
- Margins: 0.5 to 1 inch
- File: .docx or clean text-based PDF
- Bullets: 1-2 lines max, under 20 words, no periods
- 2-4 bullets per role
- Current role: Present tense, Past roles: Past tense
- Always digits not words: "3 services" not "three"
- Number first: "70% reduction" not "reduced by 70%"

## 9. NEVER INCLUDE
Photo, Age/Gender/Marital/Religion, "References available", Salary, "I/my/me", Objective statements, Unverifiable skills, Irrelevant history, Spelling errors

## 10. PRE-SUBMIT CHECKLIST
- Starts with strong unique action verb?
- Contains metric/measurable result?
- 1-2 lines max (under 20 words)?
- Keyword in first 3-4 words?
- Follows X-Y-Z formula?
- No weak verbs or empty claims?
- One page only? Single column?
- No repeated verbs?
- Soft skills shown, not listed?

You are an expert in writing personal profiles.

From this file repo-info.md, create separate profiles of your experience, categorized by type: work experience, projects, awards and competitions, skills, and by specialization (such as: Data Engineer, Flutter Developer, Systems Engineer, or any specializations relevant based on the information in this file).

Create the file: repo-info.md
Prompt 2: التحقق من النتيجة 
Review the generated profiles carefully. If anything is missing (such as skills portfolio, specific projects, or experiences), send a follow-up prompt like:

"I didn't create a skills portfolio."

Or specify exactly what needs to be added or corrected.
